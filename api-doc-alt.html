Commit

Up 
post /ws/commit/{group}/{repo}
Commit data sets (commit)
Commits data sets to repository {group}/{repo}. For this operation the client needs write access on the specified repository

Path parameters

group (required)
Path Parameter — The group of the repository
repo (required)
Path Parameter — The name of the repository

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/octet-stream

Request body

body object (required)
Body Parameter — An input stream streaming the data sets from the client to the server as byte array. See fetch response 200 for definition of the byte array

Query parameters

lastCommitId (optional)
Query Parameter — The clients last commit id before this current commit

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • text/plain

Responses

201

Returns the new commit id if the data sets were successfully committed to repository {group}/{repo}

403

No permission to WRITE on {group}/{repo}

404

Repository {group}/{repo} not found

406

Repository json schema version is not supported

409

Out of sync, if there were other commits before the current commit, but after the commit with the specified {lastCommitId}
Up

Responses

Fetch

Up 
post /ws/public/fetch/{group}/{repo}
Fetch data sets (fetch)
Fetches all data sets for repository {group}/{repo} that were added after/until the specified {commitId}. For this operation the client needs read access on the specified repository

Path parameters

group (required)
Path Parameter — The group of the repository
repo (required)
Path Parameter — The name of the repository

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body FileReference (optional)
Body Parameter — A list of data sets to fetch (clients might have identified already updated files, this can be used to specifiy which data sets actually need to be returned

Query parameters

commitId (optional)
Query Parameter — The commit id to fetch data after/until
download (optional)
Query Parameter — If true fetches all data sets until {commitId}, if false fetches all data sets after {commitId} default: false

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/octet-stream

Responses

200

Returns an output stream, streaming the collected data sets as byte array. For each data set a byte array is constructed by the following pattern: 4 bytes specifying the length of the data set descriptor, followed by the bytes of the data set descriptor json string itself. Next, 4 bytes specifying the length of the data set, followed by the bytes of the data set json string itself. If any binary data is attached, 4 bytes specifying the number of binary files. Next, for each binary file: The length of the path string, followed by the path string itself, followed by the length of the data, followed by the data itself. The length information is in Big-Endian order, the data strings are utf-8 encoded and, the data strings and binary files are also gzipped

204

No data sets to fetch

403

No permission to READ on {group}/{repo}

404

Repository {group}/{repo} not found

406

Repository json schema version is not supported

get /ws/public/fetch/request/{group}/{repo}
Request to fetch data sets (requestFetch)
Loads data set descriptors to be fetched in a subsequent "fetch" invocation. Clients can use this method to display changes to the user before invoking the actual fetch operation. For this operation the client needs read access on the specified repository

Path parameters

group (required)
Path Parameter — The group of the repository
repo (required)
Path Parameter — The name of the repository

Query parameters

lastCommitId (optional)
Query Parameter — The last fetched commit, before executing the current fetch request
sync (optional)
Query Parameter — If true includes all data set descriptors until {lastCommitId}, if false includes all data set descriptors after {lastCommitId} default: false

Return type

FetchRequestData

  • application/json
